HTTP

Syfte?

Syftet med HTTP är att möjliggöra överföring av filer över internet. Filerna överförs mellan webbklienter och webbservrar. Mer info finns i: Word Wide Web artikeln på Wikipedia

Vad gör HTTP?

HTTP protokollet fungerar så att klienten först skickar en begäran om en resurs (till exempel en HTML webbsida eller en bild) från en webbserver. Webbservern skickar då ett svar till klienten med den begärda resursen (om klienten har behörighet att visa den).

TCP

Delar upp information i små delar (paket) som sedan skickas till mottagaren med hjälp av IP protokollet nedan .

IP

IP protokollet används för att veta till vilken ip-adress och maskin en fil ska skickas. Maskiner identifieras med hjälp av ett ip-nummer.

Delarna av en URL

Example:http://medieteknik.lnu.se/1me321/uppg/u1.html#fragor

  • http:// - Protokollet som används för överföring av filerna
  • medieteknik.lnu.se/ - Domänen som webbservern finns på.
  • 1me321/uppg/u1.html - Sökvägen till resursen vi vill åt
  • #fragor - Ett "ankare" som indikerar till vilken sektion på webbsidan som ska visas när sidan laddas.
